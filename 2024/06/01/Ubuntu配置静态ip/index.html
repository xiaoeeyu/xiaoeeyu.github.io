Ubuntu配置静态ip

发表于2024-06-01|更新于2024-06-01|工具
|总字数:657|阅读时长:2分钟|浏览量:

例如要将Ubuntu虚拟机的IP地址更改为192.168.1.23，并且使用VMware的桥接模式，可以按照以下步骤进行：

环境：这里示例环境是Ubuntu22.04，在其它环境下可能也有所不同

在VMware中设置桥接模式

  1. 打开VMware并启动虚拟机： 确保你的Ubuntu虚拟机已启动并正在运行。
  2. 进入虚拟机的设置： 右键点击虚拟机，然后选择“Settings”或“虚拟机设置”。
  3. 选择网络适配器： 在设置窗口中，找到并选择“Network Adapter”（网络适配器）。
  4. 选择桥接模式： 在网络适配器设置中，选择“Bridged: Connected directly to the physical network”（桥接：直接连接到物理网络）。确保取消勾选“Replicate physical network connection state”（复制物理网络连接状态）。

在Ubuntu中设置静态IP地址

  1. 打开终端： 在Ubuntu虚拟机中打开终端。

  2. 编辑Netplan配置文件： 使用sudo权限编辑Netplan配置文件。Netplan是Ubuntu用于管理网络配置的工具。通常，配置文件位于/etc/netplan/目录下，文件名可能是01-netcfg.yaml或类似的。

    sudo nano /etc/netplan/01-netcfg.yaml

  3. 配置静态IP地址： 编辑配置文件，使其包含以下内容：

    network:
  version: 2
  ethernets:
    ens33:  # 确认接口名称是否为ens33，如果不同则修改为实际名称
      dhcp4: no
      addresses: [192.168.1.23/24]
      routes:
        - to: default
          via: 192.168.1.1  # 根据实际网关地址修改
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

    请注意：

    • ens33是网络接口的名称，可能会根据具体情况有所不同，可以使用ip a命令查看实际的接口名称。
    • gateway4设置为你的网关地址，通常是路由器的IP地址。

  4. 应用配置： 保存文件并退出编辑器，然后应用新的网络配置：

    sudo netplan apply

  5. 验证配置： 使用以下命令检查IP地址是否已成功更改：

    ip a

    确认网络接口已经获得了新的IP地址192.168.1.23。

确认网络连通性

  1. 检查连接： 在终端中使用ping命令检查是否可以连接到网关或其他网络设备：

    ping 192.168.1.1

  2. 确保互联网连接： 还可以使用ping命令检查是否可以访问互联网：

    ping baidu.com

这样，Ubuntu虚拟机的IP地址应该已经成功更改为192.168.1.23，并且连接到你的主机所在的网络上。如果在执行过程中遇到任何问题，请确保虚拟机网络适配器的设置正确，并且配置文件中的语法没有错误。
